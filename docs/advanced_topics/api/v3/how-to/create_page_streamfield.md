# How to create a page with StreamField content

This guide walks through creating a new page via the v3 API when the page model
uses a `StreamField`. All examples run against
[bakerydemo](https://github.com/wagtail/bakerydemo) with its standard fixtures.

## Prerequisites

- Wagtail v3 Write API installed and mounted (see {ref}`api_ninja`).
- bakerydemo running locally with demo fixtures loaded.
- The authenticated user has `add` permission on the target parent page.

## Background: StreamField JSON format

A StreamField value is serialised as a JSON array of **blocks**. Each block has:

| Key | Description |
|-----|-------------|
| `type` | The block's name as defined in the `StreamField` declaration |
| `value` | The block's value (varies by block type) |
| `id` | A UUID (auto-generated by Wagtail if omitted) |

The bakerydemo `BreadPage` model has a `body` `StreamField` with `heading`,
`paragraph`, `image`, and `quote` blocks. A minimal valid body looks like:

```json
[
  {
    "type": "heading",
    "value": "My new bread heading"
  },
  {
    "type": "paragraph",
    "value": "<p>Fresh from the oven, via the API.</p>"
  }
]
```

## Creating the page

Use Django Ninja's `Schema` to validate the incoming payload and Wagtail's
`Page.add_child()` (treebeard's tree API) to attach the new page to its parent:

```python
# api.py
import json
from typing import Any

from ninja import Router, Schema
from ninja.errors import HttpError
from wagtail.models import Page
from wagtail.permission_policies.pages import PagePermissionPolicy

from breads.models import BreadPage

router = Router()
page_policy = PagePermissionPolicy()


class StreamBlockIn(Schema):
    type: str
    value: Any
    id: str | None = None


class BreadPageIn(Schema):
    title: str
    slug: str
    introduction: str = ""
    # StreamField body – list of block dicts
    body: list[StreamBlockIn]
    parent_page_id: int


@router.post("/pages/breads/")
def create_bread_page(request, payload: BreadPageIn):
    """Create a new BreadPage under the given parent page."""
    parent = Page.objects.get(pk=payload.parent_page_id)

    # Permission check: user must be able to add under this parent.
    if not page_policy.user_has_permission_for_instance(request.user, "add", parent):
        raise HttpError(403, "You do not have permission to add pages here.")

    # StreamField stores a JSON string, not a Python list.
    body_json = json.dumps([b.dict() for b in payload.body])

    page = BreadPage(
        title=payload.title,
        slug=payload.slug,
        introduction=payload.introduction,
        body=body_json,
    )

    # add_child saves the page and inserts it into the page tree.
    parent.add_child(instance=page)
    return {"id": page.pk, "title": page.title}
```

## Example request

In bakerydemo the "Breads" index page has pk `5` in the standard fixtures:

```sh
curl -X POST http://localhost:8000/api/v3/pages/breads/ \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Sourdough Rye",
    "slug": "sourdough-rye",
    "introduction": "A classic tangy loaf.",
    "parent_page_id": 5,
    "body": [
      {"type": "heading", "value": "About this bread"},
      {"type": "paragraph", "value": "<p>Sourdough rye is made with…</p>"}
    ]
  }'
```

Expected response:

```json
{"id": 42, "title": "Sourdough Rye"}
```

## Working with nested StructBlocks

For blocks that contain nested data (e.g. bakerydemo's `quote` StructBlock),
the `value` should be a dict matching the struct's field names:

```json
{
  "type": "quote",
  "value": {
    "quote": "Good bread is the most fundamentally satisfying of all foods.",
    "attribution": "James Beard"
  }
}
```

## Verifying the page was created

```sh
# Fetch the new page from the v2 read API (no auth required for public pages)
curl http://localhost:8000/api/v2/pages/42/?fields=title,slug,body
```

## Accessing StreamField in templates

When referencing StreamField values in Wagtail templates, use the block type:

```html+django
{% for block in page.body %}
  {% if block.block_type == 'paragraph' %}
    <div class="paragraph">{{ block.value }}</div>
  {% elif block.block_type == 'heading' %}
    <h2>{{ block.value }}</h2>
  {% endif %}
{% endfor %}
```

```{note}
The `block.value` property gives the rendered output for simple blocks.
For `StructBlock` or `ListBlock`, `block.value` is itself iterable.
See [](../../../../extending/custom_streamfield_blocks) for custom block types.
```
